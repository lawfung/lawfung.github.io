Github pages 建立筆記

我使用 Hexo 來建立 Github pages。

建議先閱讀資料

  • 首先你需要知道甚麼是 Github pages
  • 然後你需要知道甚麼是 Hexo
    • 大概先把 commands 那節以前(含)讀完，自己在 local 跑一下 server 熟悉一下
  • 然後你會看到 github-pages 這節，我所使用的是One-command-deployment這個部分
  • 你最好已經寫過 Markdown (比如說使用Hackmd)

手把手教學

看著這個照做，裡面有從下載Hexo到One-command-deployment的完整指令。

小提示

  • 裡面每一步提供的連結的內文最好還是都看一下
  • (For step 10) hexo deploy 差不多就是把 commit 和 push 全包了。需要注意的是，如果你原本 username.github.io 這個 repo 原先就已經有了 commit ，那麼第一次 deploy 會把原先的 commit 全部都蓋掉。
  • (For step 10) 如果要預覽的話可以用 hexo server 指令，可以不要一直 deploy ， commit 歷史會變很醜
    • 更改文章內容的話，刷新頁面就可以了，但是如果改 _config.yml ，就要重開 hexo server
    • 如果改了 _config.yml ，覺得新頁面怪怪的，可以試用 hexo clean 再開 hexo server。
  • (For step 11) 這裡有很多主題可以選

關於文章

  • 這邊跟 Hackmd 不一樣的是沒有內建 latex 。如果想寫數學式，可以看著這個照做。大概會像是
  • 沒有刪除文章的指令，但是可以直接把檔案刪掉
  • 文章的網址是 source/_posts 裡面的檔名決定的
